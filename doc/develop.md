# 开发者指南

## 环境依赖
> 本项目全部由 `python` 完成开发，开发版本为 `3.12.4` 

1. 安装虚拟环境（可选）
```bash
python -m pip install virtualenv
python -m venv .env
```

2. 安装库依赖
```bash
python -m pip install -r requirements.txt
```

3. 编译或者直接运行
```bash
python -m pip install pyinstaller
python -m pyinstaller -F ./src/main.py
```

## 相关原理解释

1. 读取相关规则（`./settings/rule.yaml`），对每一个人建立一个无向图（`dict`）该映射关系将决定每个人可能可以坐同桌的人。
2. 将不同组的人分开处理，并将有白名单的人直接连接，有黑名单的跳过其黑名单的人。
3. 该问题实际上就变成了对于一个无向图，将其分为大小不大于 $k$ 的连通块，使联通块数量最小。
4. 考虑 $k=2$ 的特例，注意到该情况等效于选出一些边使得任意两条边不相交（没有公共顶点），即 `无向图最大匹配`
5. 对于这个问题，有一个巧妙的解法。
   1. 将无向图的每个顶点拆成两个点：出点和入点。
   2. 对于原图的每条从 $u$ 到 $v$ 的边，我们将其变为从 $u$ 的出点到 $v$ 的入点的边。
   3. 注意到经过这个转化后，原图变成了一个二分图（即出点的点集与入点的点集内部没有边相连）。而原问题转变为在新图中取出一些对应关系使得每一组关系不重复，即 `二分图最大匹配`。
   4. 我们建立两个点 $s$ 和 $t$ ，将 $s$ 向所有的出点连权值为 $1$ 的边，所有的入点向 $t$ 连权值为 $1$ 的边，则可以用 `最大流最小割定理` 和 `Dinic 算法` 完成这个问题。时间复杂度：$\Omicron(n \sqrt m) = \Omicron(n^2)$
6. 对于 $k \ne 2$ 的情况，将总人数用虚点补全到 $2\times kt$ 然后套用 $k = 2$ 的方法。生成完成后从 $kt$ 组中随机选择 $t$ 对 $k$ 组，将其随机合并并且分为两组即可。


## 参与贡献
1. 先 **fork** 到自己的仓库。 
2. 仓库分支大致为 `master-dev-feat-fix` 的结构，如果有开发新功能的想法，就从 `dev` 或者 `master` 上分出 `feat-xxxx` （`xxxx`为功能名）分支；类似的，如果有修 bug 的想法，就分出 `fix-xxxx` 分支。
3. 开发完成后，将新添加的分支合并到 `dev` 分支并发起 `Pull Request`。
